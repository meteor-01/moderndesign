### 抽象工厂模式

<strong>描述：提供一个创建一系列相关或相互依赖对象的接口</strong>

抽象工厂模式和工厂方法模式最大的区别就是抽象工厂模式多了一个超级工厂，超级工厂的作用是对不同的方法进行抽象，然后将相同类型的实现放到同一个工厂中进行创建，达到减少创建工厂的目的，比如小米手机和华为手机属于不同类型手机厂家的不同实现，而华为手机和华为电脑属于相同类型厂家的不同实现

对于示例代码，工厂方法模式给每个Service都创建了一个工厂用以创建具体类，最后创建了八个工厂类，可见，随着功能增加工厂类会越来越多，抽象工厂模式则将方法进行划分，相同类型的方法进行抽象，大幅度减少了工厂的创建，示例代码中抽象出CloudService和QueryService两个接口，所有属于云服务的都实现CloudService接口，属于咨询服务的都实现QueryService接口，此时有AliFactory和ByteFactory两个工厂，如果增加了新的Servic，那么只需要在这两个工厂添加新的方法即可，但如果增加了新的工厂，那就需要增加很多方法和子类，这也是抽象工厂模式的缺点
